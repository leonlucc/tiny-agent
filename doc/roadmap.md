# 渐进式演进路线图 (Full Roadmap)

Tiny Agent 坚信“最好的学习方式是亲手构建”。为了避免一上来就面对复杂的完整系统，本项目将 Agent 的核心能力拆解到不同的 Git Tag 中。

> 注：你可以通过 `git checkout` 命令切换到任意版本进行独立运行和源码研读。如 `git checkout v0.3.0` 将切换到 RAG 版本。

## [v0.0.1] Hello LLM（你好，大模型）

* **定位**：项目起步，跑通最基础的闭环。
* **核心能力**：学习如何配置和初始化大语言模型（LLM）SDK，实现单轮文本输入与输出。
* **UI演进**：无前端 UI，纯 CLI 终端交互，方便开发者聚焦后端网络请求本身。

## [v0.0.2] Streaming Web（流式 Web 对话）

* **定位**：从 CLI 迈向实时响应的浏览器交互。
* **核心能力**：搭建最小后端服务与前端页面，通过 SSE 协议实现 LLM 流式输出的实时渲染。
* **UI演进**：引入极简 Web页面 （输入框 + 发送按钮 + 结果展示区），动态解析并逐字打印流式文本。

## [v0.1.0] Chat（基础对话）

* **定位**：从“一问一答”走向“连续对话”。
* **核心能力**：理解 Message Protocol 的设计，设计 Chat History 数据结构，实现多轮连续对话。
* **UI演进**：增加消息列表，可保留历史记录，支持连续追问。

## [v0.2.0] Prompt Engineering（提示词工程）

* **定位**：让模型听话，按规矩办事。
* **核心能力**：引入 System Prompt、Structured Outputs、JSON Schema 等现代 LLM 交互方式，为后续 Tool Calling 做准备。
* **UI演进**：侧边栏增加配置面板，支持动态修改 System Prompt、Temperature 等模型参数。

## [v0.3.0] Basic RAG（基础检索增强生成）

* **定位**：为模型提供可检索的外部知识库，降低无依据生成的概率。
* **核心能力**：对接 Embedding 模型，引入轻量级向量库，实现文档解析、文本切分、向量化与本地检索，打通「切片 → 检索 → Prompt 拼装 → 生成」的最简闭环链路。
* **UI演进**：增加知识库管理模块，支持前端拖拽上传文档，并直观展示切片后的文本块；对话框中支持显示文档引用来源。

## [v0.4.0] Tool Calling（工具调用）

* **定位**：赋予 Agent 改变世界、连接外部系统的双手。
* **核心能力**：深入 Function Calling 原理，学习工具声明、参数生成、本地函数执行以及工具结果回填，理解 LLM 如何调用外部能力；加入最小安全机制。
* **UI演进**：在对话流中高亮展示 Tool 的调用状态与链路（参数 → 调用状态 → 返回结果），工具执行时输出安全提示。

## [v0.5.0] MCP（模型上下文协议）

* **定位**：从项目私有工具接口走向标准化能力协议。
* **核心能力**：理解 MCP（Model Context Protocol）的设计思想，实现 MCP Client 对接，学习 Tool Discovery以及 Client-Server 工具连接模型，掌握本地工具与 MCP 工具的统一接入方式。
* **UI演进**：新增 MCP Server 管理页面，支持连接、断开和查看已加载的远程工具列表。

## [v0.6.0] Workflow（工作流）

* **定位**：从单次工具调用走向可编排、可复用的任务流程。
* **核心能力**：理解 AI Workflow 的设计思想，学习任务拆解、节点编排、状态流转与流程控制，构建由多个步骤组成的工作流，为后续 Agent 的自主规划奠定基础。
* **UI演进**：新增 Workflow 简易界面，展示节点之间的执行顺序及运行状态。

## [v0.7.0] Agent Loop（智能体循环）

* **定位**：从被动调用工具到自主思考（Agent 的智能核心）。
* **核心能力**：手写 ReAct 范式核心循环，让 Agent 学会拆解复杂任务，自主进行 Thought → Action → Observation 状态循环直至完成目标，并加入基础的最大循环步数、总Token上限机制。理解确定性工作流与非确定性自主循环的边界与协同。
* **UI演进**：动态可视化 Agent 的执行轨迹，以状态卡片展示计划摘要、工具调用、观察结果与任务进度。

## [v0.8.0] Evaluation & Tracing（评估与链路追踪）

* **定位**：让 Agent 从“看起来能运行”走向“效果可验证、过程可追踪”。
* **核心能力**：建立最小评估集，记录任务完成率、工具调用准确率、检索命中率、Token 消耗与执行耗时；为每次运行生成统一 Trace，串联模型请求、检索结果、工具调用、状态变化、错误与停止原因，并支持执行链路回放和版本回归对比。
* **UI演进**：新增 Trace 调试页面，以时间线方式展示完整执行过程，并提供基础指标面板，支持按模型、Prompt、任务类型和运行结果筛选记录。

## [v0.9.0] Agent Runtime（智能体运行时）

* **定位**：为 Agent 提供统一、可靠且可控制的执行环境，使其从“能够完成任务”走向“能够稳定运行”。
* **核心能力**：抽象统一的 Agent Runtime，负责管理任务生命周期与运行状态；实现状态持久化、超时控制、主动取消与基础重试机制；统一调度 Workflow、Agent Loop、Tool 与 MCP 调用，并定义运行中、已完成、已取消、超时和失败等标准状态，支持任务中断后的恢复与资源清理。
* **UI演进**：增加运行控制能力，支持查看任务当前状态，并提供取消、重试和恢复等基础操作。

## [v1.0.0] Tiny Agent Core（极简智能体）

* **定位**：核心里程碑，标准 Agent 体系初步达成。
* **核心能力**：将 LLM + RAG + Tool Calling + MCP + Workflow + Agent Loop 深度整合，形成高内聚、低耦合的单体 Agent 架构。
* **UI演进**：打通所有模块，交付全功能 Agent 综合控制面板，兼具调试与日常使用属性。

## [v1.1.0] Skills (技能)

* **定位**：让 Agent 的能力实现模块化管理。
* **核心能力**：引入 Skills 概念，实现“技能包”的动态注册、能力封装、按需加载、动态发现以及自动路由，理解 Skill 与 Tool 的职责边界，构建可扩展的能力体系。
* **UI演进**：提供 Skills 管理界面，支持启用 / 禁用技能，查看技能包含的工具及调用统计。

## [v1.2.0] Context Management (上下文管理)

* **定位**：给 Agent 装上“工作内存”，解决当前任务循环中的上下文失控问题。
* **核心能力**：设计Token 预算熔断机制，实现滑动窗口裁剪（Sliding Window）与即时会话压缩（Conversation Summary）。
* **UI演进**：对话框顶部增加 Token 实时温度条；增加熔断卡片提示。

## [v1.3.0] Long-Term Memory (长期记忆)

* **定位**：打破会话隔离与 Token 窗口限制。
* **核心能力**：设计记忆漏斗模型，实现记忆提取 → 价值判断 → 去重 → 写入 → 检索 → 注入 → 更新 → 遗忘机制。
* **UI演进**：提供 Memory 管理页面，允许开发者查看、检索甚至手动编辑 Agent 的深层记忆。

## [v1.4.0] Advanced RAG（进阶检索增强生成）

* **定位**：工程化能力进阶，攻克真实场景下“检索不准、上下文断裂”的硬伤。
* **核心能力**：实现语义检索与关键词检索（BM25）的混合检索（Hybrid Search）；引入 Rerank（重排）机制提升 Top-K 精度；引入生产级向量数据库（如 Qdrant / PGVector）；将 RAG 封装为 Agent 可自主调用的工具，动态决策。
* **UI演进**：升级知识库面板，支持混合检索和重排配置。

## [v1.5.0] Security & Permissions（安全与权限）

* **定位**：为 Agent 增加安全边界，确保工具调用可控、可审计。
* **核心能力**：建立工具权限分级，引入参数校验、调用白名单、与用户确认机制，防止越权调用和高风险操作。
* **UI演进**：新增审计页面，高风险操作执行前展示确认卡片。

## [v1.6.0] Voice Interaction（语音交互）

* **定位**：让 Agent 具备语音输入与输出能力。
* **核心能力**：对接 ASR 与 TTS 服务，实现语音实时转写与流式语音合成；加入语音打断机制，语音通路与现有文本对话管线无缝融合。
* **UI演进**：新增语音录制按钮与波形可视化，消息列表支持语音回放。

## [v1.7.0] Multimodal Vision（多模态视觉）

* **定位**：赋予 Agent 多模态视觉能力。
* **核心能力**：对接 Vision LLM 实现图片理解，接入图像生成模型；扩展消息协议以支持图片类型；支持通过工具调用触发图片生成。
* **UI演进**：输入区增加图片上传/粘贴，对话流支持图文混排与点击放大，生成过程显示加载状态。

## [v2.0.0] Multi-Agent (多智能体)

* **定位**：探索多智能体协作模式。
* **核心能力**：定义不同角色的子 Agent，手写任务分发与状态路由，实现智能体之间的静态、动态协作与监督机制，并理解 Multi-Agent 的适用场景与工程边界。
* **UI演进**：提供 Agent 管理面板，支持查看和配置各 Agent 的角色、状态及协作关系。
